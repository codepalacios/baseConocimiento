Fundamentos del Desarrollador Web

Existen tres estándares importantes para generar productos web. Esto son los lenguajes o estándares que entiende todo navegador web:

  • HTML: estructura de los documentos que se visualizan en los navegadores (Chrome, Edge, Firefox, Safari, Opera, entre otros).
  • CSS: es el lenguaje con el que se describen estilos a todo el contenido previamente etiquetado con HTML.
  • JavaScript: es el lenguaje que se emplea para crear o brindad a las páginas web la capacidad de interactuar con el usuaro y los recursos asociados a una página web.
¿Qué es el Frontend?

Frontend es la parte de un programa o dispositivo a la que un usuario puede acceder directamente. Son todas las tecnologías de diseño y desarrollo que corren en el navegador y que se encargan de la interactividad con los usuarios.

Un programador Frontend debe saber de código que entienda el navegador (HTML, CSS y JavaScript) para poder usar algunos frameworks o librerías que expanden sus capacidades para crear cualquier tipo de interfaces de usuarios.

¿Qué es Backend?

Backend en programación corresponde al lado opuesto a un Front-end en un sitio web o aplicación, ya que el Backend trabaja en el lado del servidor. Es el responsable de manejar toda la lógica que hay detrás de una petición dada por el navegador hacia el servidor.

Una carecterística que lo diferencia del Frontend es que no tiene estándares, puesto que tiene varios lenguajes de programación (Node.js, Python, PHP, Ruby, GO, Java, .NET entre otros) con los que debe trabajar. Cada uno de estos lenguajes tiene su propios frameworks como:

  • Django (Python)
  • Laravel (PHP)
  • Rails (Ruby)
  • Express (JavaScript)
  • Spring (Java)

El Backend también tiene en cuenta la infraestructura donde va a realizar el deploy de su aplicación (esto también puede ser tarea de un DevOps, un perfil dedicado a la infraestructura), con tecnologías como:

  • Google Cloud
  • DigitalOcena
  • AWS
  • Heroku, entre otras.

¿Qué es deploy?

Deploy es un término famoso entre los desarrolladores web. Puede significar muchas cosas, dependiendo del ambiente y de la tecnología usada. Sin embargo, los significados que más se refieren a la práctica y pueden resumir su función son: implantar, colocar en posición, habilitar para uso o, simplemente, publicar.

Por último, entramos en base de datos, que son las encargadas de almacenar toda la información del proyecto Los principales tipos son:

  • Bases de datos relacionales (como MySQL)
  • Base de datos no relacionales (como mongoDB)
¿Qué es Full Stack?

Full Stack es un término utilizado para describir a los desarrolladores que conocen tanto los lenguajes de frontend como de backend. Principalmente, el desarrollador full stack se refiere al uso de JavaScript en el backend y de HTML/CSS/JavaScript en el frontend para crear nuevas plataformas.

El nacimiento de tecnologías que funcionan entre el frontend y el backend ha dado lugar a la proliferación de frameworks y herramientas de desarrollo full stack, que permiten a los desarrolladores construir sus propias aplicaciones web completas empleando un único lenguaje de programación, como Django para Python.

¿Qué es un desarrollador Full Stack?

Las y los desarrolladores Full Stack son profesionales que se encargan tanto de la parte visual y de la interaccion de un sitio (frontend), como de su lógica y funcionamiento del lado del servidor (backend).

Un Stack (en inglés: pila o montón) se refiere al grupo de tecnologías que componene un sitio web en todos los aspectos (desde la base de datos, hacia el manejo lógico y la interfaz visual). Una o un desarrolador Full Stack en teoría es capaz de manejar la pila completa de un sitio, tanto de frontend como de backend, además saber utilizar su base de datos.

¿Qué hace un desarrollador Full Stack?

Una desarrolladora o desarrollador Full Stack entiende muy bien cómo funciona un producto web de principio a fin, desde su diseño en mockup y deploy hasta producción.

Este tipo de programador o programadora no maneja por completo todas las tecnologías de ambas partes, pues cada una requiere conocimiento profundo. De hecho, no es recomendado profundizar en tantas especialidades y no es sano. El desarrollo web evoluciona muy rápido y cada dos o tres meses aparecerá algo nuevo.

¿Cómo ser un desarrollador Full Stack?

Para convertirse en Full Stack developer se debe dominar las dos áreas básicas: backend y frontend. Esto ayudará a comprender cómo funciona cada uno y a generar una comunicación entre sí. La mayoría de la interacción entre estos dos mundos se basa en consumo de API y comprender cómo consumir una API desde el Frontend ayudará a entender cómo construir una API desde el Backend.

Además, aprender las diferentes formas de subir en internet una aplicación web ayudará a entender las diferentes estrategias de deploy que existen y permitirá ampliar los conocimientos en el mundo del desarrollo web.

Motores de renderizado

La función principal es pasar de archivos a píxeles en pantalla. Los navegadores tienen motores y todos ellos básicamente hacen lo mismo para pasar los archivos a píxeles.

  • Chrome (Blink)
  • Edge (Edge HTML)
  • Safari (Webkit)
  • Firefox (Gecko)

El navergador hace 5 pasos:

  1. Pasa los archivos a objetos (DOM)
  2. Calcula el estilo correspondiente a cada nodo DOM
  3. Calcula las dimensiones de cada nodo y dónde va en la pantalla
  4. Pinta las diferentes cajas
  5. Toma las capas y las convierte en una imagen para mostrar en pantalla
Páginas Estáticas vs. Dinámicas

Los sitios web se comportan de forma diferente dependiendo de la forma en que fueron diseñados desde su concepción, tomando en cuenta la interacción con el usuario.

Sitios Web Estáticos

La información que contiene se mantiene constante y estática. No se actualiza con la interacción del usuario. Es conveniente para realizar landing pages (páginas informativas o de aterrizaje) o blogs. Se mostrarán siempre iguales para todos los usuarios.

Ejemplo de páginas estáticas:

  • Menú de un restaurante
  • Blog de viajes
  • Página informativa de un negocio

Sitios Web Dinámicos

También conocidos como aplicaciones web, actualizan su información con respecto a la interacción del usuario. Dependen de una base de datos, de donde extrae e ingresa información. Serán diferentes, dependiendo del usuario que la use y la información que se ingrese.

Ejemplo de páginas dinámicas:

  • Sistema de reporte de ventas
  • Linkedin
  • Banca en línea
Anatomía de un elemento HTML

Elemento y Atributo

Un elemento HTML está compuesto por una etiqueta de apertura, el contenido y una etiqueta de cierre. Adicionalmente puede contener uno o más atributos que ayudan a modificar el comportamiento del elemento.

Anidamiento

Las etiquetas anidadas son aquellas que se encuentran entre la instrucción de inicio y la instrucción de fin de otra etiqueta. De esta forma conseguimos que los efectos de ambas se sumen. En otras palablas anidar una etiqueta es lo mismo que meterla dentro de otra.

Elementos vacíos

Un elemento vacío es un elemento HTML que no puede tener nodos secundarios (es decir, elementos anidados o nodos de texto).

Anatomía de un documento HTML

La estructura básica de una página web se compone por las etiquetas <html>, <head> y <body>. La primera indica el comienzo de la página; la segunda es la cabera y contiene información descriptiva del sitio; y la tercera es la que contiene todos los elementos visibles dentro de una página.

La etiqueta <body> identifica la parte visible del sitio web. Dentro del body, se ańadirán las etiquetas para marcar los elementos visuales del sitio web, como logotipo, menús de navegación, contenido principal, entre otros. Es muy importante usar HTML semántico y no llenar todo de <div> para que los sitio web sean mejor interpretados por el navegador y, por lo tanto, más accesibles.

¿Qué es HTML semántico?

HTML semántico es el uso de etiquetas con significado que ayudan a un sitio web a ser accesible, mejorando el posicionamiento SEO (Posicionamiento en Buscadores) y tener un código más claro.

Las etiquetas en HTML ayudan al desarrollador y usuario a diferencia en qué parte del contenido se encuentra. HTML son las siglas que corresponden a HyperText Markup Languaje (Lenguaje de Marcado de Hipertexto).

  • Hyper Text significa que el texto tiene interactividad, conexión con otros documentos.
  • Markup significa que le asigna etiquetas a los elementos. Por eso también se le conoce como un lenguaje de etiquetas.
  • HTML es un lenguaje interpretado. Además, HTML es un estándar, así que no importa desde qué navegador o dispositivo se ejecute, el código sigue siendo el mismo en cualquier sitio.

La web se conforma de tres conceptos:

  • URL: Uniform Resource Locator. El identificador único del sitio en el navegador (por ejemplo: https://platzi.com).
  • HTTP: Protocolo de transferencia de hipertexto. Es el estándar que se utiliza para enviar datos a través de paquetes entre el cliente y el servidor.
  • HTML: es el código que se emplea para estrucutrar el contenido de una web, darle sentido y propósito.

Etiquetas de HTML más usadas

Existen una amplia variedad de etiquetas en HTML, pero ciertas etiquetas presentan un uso más recurrente por parte de los desarrolladores:

  • Layout: header, nabvar, section, article, aside, footer, div.
  • Textos: h1...h6, p, span.
  • Imágenes y video: img, svg, iframe, video.
  • Formulario: form, input, label, button.
  • Listas: ul, li, ol.
  • Enlaces: a (ancla).
Imágenes para la web

Las imágenes representan una pieza fundamental al momento de mostrar contenido para la web. El tamaño máximo recomendado para una imágen es 70kb. También es importante conocer los principales tipos de imágenes web y sus formatos.

Tipos de imágenes para la web

Lossless (sin pérdida):

  • Captura todos los datos del archivo original.
  • No se pieder nada del archivo original.
  • Puede comprimirse, pero podrá reconstruir su imagen al estado original.

Lossy (con pédida):

  • Se aproxima a su imagen original.
  • Podría reducir la cantidad de colores en su imagen o analizar la imagen en busca de datos innecesarios.
  • Por consiguiente puede reducir su tamańo, lo que mejora el tiempo de carga de la página, pero pierde su calidad.
  • Los archivos tipo lossy son mucho más livianos que los archivos tipo lossless, por lo que son ideales para usar en sitios en doden el tamaño del archivo y la velocidad de descarga son importantes

Formatos de imagen para web

  • GIF (Graphics Interchange Format): formato de imagen si pérdida, no se puede comprimir.
  • PNG 8 (Portable Network Graphics): formato de imagen si pédida, uso de colores de 256, se utiliza para logotipos e iconos para la página.
  • PNG 24 (Portable Network Graphics): formato de imagen sin pédida, utilización de colores ilimitados, alta calidad, si intentamos comprimir no ayudará demasiado por la gran cantidad de colores.
  • JPG / JEPG (Photographic Experts Group): formato de imagen con pédida, perderemos calidad a la hora de comprimirlas, pero llegan a ser óptimas para la carga en la página web.
  • SVG - Vector (Scalable Vector Graphics): formato de imagen muy ligero sin pérdida, con svg no perderemos calidad, ya que está compuesta por vectores.
  • WebP: es un formato gráfico en forma de contenedor que sustenta tanto compresión con pérdida como sin ella. Fue desarrollado por Google.

Selectores de pseudoclases y pseudoelementos

Las pseudoclases permiten llegar a las actividades o acciones que realiza el usuario, por ejemplo: darle clic a un botón, pasar el mouse sobre una imagen, botón o texto; para poder estilizarlo. Los pseudoelementos permiten acceder a los elementos de HTML que no son accesibles por otros selectores, por ejemplo: la primera letra de un texto, agregar contenido antes o depués de un texto, entre otros; para poder estilizarlo. Ambos se diferencian por que las pseudoclases se escriben con dos puntos y los pseudoelementos con dobles dos puntos.

Cascada y especificidad en CSS

Hojas de Estilo en Cascada (del inglés Cascading Style Sheets) o CSS es el lenguaje de estilos utilizado para describir la presentación de documentos HTML. Cascada significa que el orden de las reglas en CSS, importa. Los navegadores determinan que reglas de CSS deben aplicarse a las etiquetas HTML. El orden y el origen de clasificación del CSS, intervienen en el proceso. Sin embargo, no todo es color de rosa. A veces los navegadores entran en conflicto porque no saben que regla aplicar. La especificidad de los selectores es la manera que tiene CSS de lidiar con los conflictos. Entre mayor especificidad, mayor va a ser la probabilidad de que sus declaraciones se usen sobre las demás.

Tipos de display más usados

Display es el tipo de visualización (el tipo de cuadro de representación) que tienen los elementos.

  • Block: muestra un elemento como un elemento de bloque (como <p>). Comienza en una nueva línea y ocupa todo el ancho.
  • Inline: muestra un elemento como un elemento en línea (como <span>). Cualquier propiedad de alto y ancho no tendrá efecto.
  • Inline-block: muestra un elemento como un contenedor de bloques de nivel en línea. El elemento en sí está formateado como un elemento en línea, pero puede aplicar valores de alto y ancho.
  • Flex: muestra un elemento como un contenedor flexible a nivel de bloque.
  • Grid: muestra un elemento como un contenedor de cuadrícula a nivel de bloque.
Modelo de cajas

Todo en CSS tiene una caja alrededor, y comprender estas cajas es clave para poder crear diseños con CSS o para alinear elementos con otros elementos. En CSS, en general, hay dos tipos de cajas: cajas en bloque y cajas en línea. El modelo de cajas CSS completo se aplica a cajas que representan comportamiento en bloque; las cajas con comportamiento en línea solo usan una parte del comportamiento definido en el modelo de cajas. El modelo define cómo funcionan juntas las diferentes partes de una caja (margen, border, relleno y contenido) para crear una caja que se pueda ver en una página web.
