Introducción a JavaScript

¿Qué es JavaScript?

JavaScript es un lenguaje de programación dinámico que tiene la capacidad de ser utilizado en muchos dispositivos diferentes. Puede usarse en ordenadores personales, servidores web y teléfonos intelegintes. Es un lenguaje interpretado, orientado a objetos, débilmente tipado y dinámico.

JavaScript se emplea comúnmente para el desarrollo web front-end y más recientemente para algunos desarrollos back-end a través de frameworks como Node.js o Next.js. Tiene características como la programación orientada a objetos, funciones y herencia basada en prototipos.

¿Cómo nace JavaScript?

Nace con la necesidad de generar dinamismo en las páginas web y que a su vez tanto los usuarios como las empresas pudieran interactuar unos con otros. Fue creado por Brendan Eich en 1995 y se convirtio en un estándar oficial del World Wide Web Consortium (W3C) en 1997.

¿Por qué JavaScript es un lenguaje dinámico?

JavaScript corre directamente en la etapa de Runtime, sin una etapa de compilación previa. Esto permite probar el código inmediatamente; pero también es lo que hace que los errores no se muestren sino hasta que se ejecuta el programa. Lo que se ve a primera vista, cuando se analiza el código, es muy probable que no sea lo que se va a obtener cuando el programa se ejecute.

JavaScript permite declarar (por ejemplo) variables cuyo valor (y tipo) solo se conocerá al momento de su ejecución en función de las condiciones dadas al momento de correrlo en un entorno real. En cambio, los lenguajes estáticos no compilarán en código ejecutable a menos que todos los valores (o tipos de valores) se conozcan por adelantado.

¿Por qué JavaScript es débilemente tipado?

Por que los tipos de datos no están bien definidos en el lenguaje y permite, por ejemplo, operaciones entre números y letras. Esto sucede porque el lenguaje asume tipos de datos que no necesariamente fueron los que se querían representar. ¿Es JavaScript un lenguaje interpretado?

Sí, y la razón es que el navegador lee línea por línea el código, el cual indica lo que tiene que ir haciendo, sin la necesidad de compilar. Todo esto es controlado por el motor de JavaScript V8 del navegador.

¿Qué significa que JavaScript es Backwards Compatible?

Todas las funciones nuevas que se agregan para JavaScript no dañarán el trabajo ya hecho previamente, pero no se prodrá utilizar en el entorno de trabajo inmediatamente. Para solucionar esto está Babel, que permite usar las nuevas características del lenguaje, pero lo transforma a una versión que el navegador pueda entender.

¿Por qué JavaScript?

  1. JavaScript tiene una comunidad enorme de desarrolladores que pueden ayudar a generar diferentes cosas.
  2. Si se desea solo trabajar en aplicaciones web existen muchos frameworks y librerías contruidas en JavaScript que ayudan a hacer proyectos de forma mucho más rápida, eficiente y robusta (Angular, View, React, entre otros).
  3. Si no desea trabajar solo en aplicaciones web se puede utilizar JavaScript con un framework que se llama React Native para poder construir apliaciones nativas para Android e IOS.
  4. Se puede construir aplicaciones de escritorio con JavaScript, usando un framework llamado Electron, pueden correr en Mac o Windows.
  5. También se puede trabajar en la parte del Back-end o IOT (Internet Od Things). Internet de las cosas es un concepto que se refiere a una interconexion digital de objetos cotidianos con Internet. Esto con un Framework llamado Node.js, el cual es un entorno de ejecución de JavaScript que corre directamente en el Back-end.

Tipos de datos en JavaScript

Los elementos que se utilizan en un lenguaje de programación son: data que se guardan en memoria y tareas (funciones) que se realizaran con esa data.

Tipos primitivos:

  • String: cadena de texto.
  • Number: valores numéricos.
  • Boolean: true ó false.
  • Null: tipo especial, contiene null.
  • Undefined: tipo especial, contiene undefined.

Tipos objetos:

  • Tipos predefinidos de JavaScript: Date (fechas), RegExp (expresiones regulares), Error (datos de error).
  • Tipos definidos por el programador / usuario: funciones simples y clases.
  • Arrays: serie de elementos o formación tipo vector o matriz. Se considera un objeto especial que carece de métodos.
  • Objetos especiales: objeto global, objeto prototipo, otros.

Operador typeof

El operador typeof devuelve una cadena que indica el tipo de valor o dato del operando en JavaScript.

¿Qué es una variable en JavaScript?

Una variable puede considerarse como un contenedor donde se puede almacenar datos, una vez guardados se pueden acceder a ellos a través del nombre de variable que se haya asignado. Dentro de JavaScript existen tres formas de declarar una variable: var, const (constante) y let.

Buenas prácticas para la declaración de variables:

  1. El nombre de la variable debe ser un nombre descriptivo que se pueda identificar con el código.
  2. Los nombres de variables no se pueden iniciar con números.
  3. No se pueden usar caracteres especiales, ni es recomendable empezar una variable el nombre de una variable con mayúscula.
  4. Los tipos de escritura de una variable son: lowerCamelCase, UpperCamelCase y snake_case.
  5. Se debe tener en cuenta que existen nombres que no se pueden utilizar porque se encuentran como palabras reservadas para el lenguaje.

¿Qué es una función en JavaScript?

Una función es un trozo o bloque de código reutilizable en el que hay un conjunto de instrucciones, dicho código solo se ejecuta cuando se llama o invoca dicha función. Las funciones pueden utilizar o no parámetros, que son los datos necesarios que se envían al llamar una función según el caso para la correcta ejecución del mismo.

Existen dos tipos de funciones: declarativas y expresivas. Las funciones declarativas se les declara con la palabra reservada function y las funciones expresivas o anónimas consisten en guardar la función en una variable. Ambas tienen la misma declaración e invocación, pero la diferencia consiste en no especificar un nombre en la función expresiva, sino que utilizar el nombre de la variable.

Bases de JavaScript

Scope

El scope es el alcance que una variable tiene en el código, decide a qué variables tiene acceso cada parte del código desarrollado. Existen dos tipos de scope, el escope global y el scope local.

El scope local permite acceder a una variable únicamente en cierta parte del código y la variable es conocida como una variable local. Un ejemplo es cuando una variable está declarada dentro de un bloque o función y solo puede ser invocada por el mismo bloque o función

En el scope global la variable está declarada fuera de un bloque o función y es conocida como variable global. Puede ser accedida desde cualquier parte del código desarrollado, ya sea dentro y fuera de un bloque o función.

Las variables declaradas con let o const dentro de un bloque, el alcance o scope que tendrán será solo dentro de ese bloque. Sin embargo, las variables declaradas con var, su valor se puede sobreescribir. Motivo por el cual no es recomendable utilizar este tipo de variables, ya que puede generar bugs.

También es importante saber que en el caso se tenga una función dentro de otra, la función externa no tiene acceso a las variables de la función interna, pero la función interna si tiene acceso a las variables de la función externa.

Hoisting

Hoisting es un término para describir que las declaraciones de variables y funciones son desplazadas a la parte superior del scope más cercano, scope global o local. Esto sucede solamente con las declaraciones y no con las asignaciones.

El código permanece igual, solo es una interpretación del motor de JavaScript. En el caso de las variables solamente sucede cuando son declaradas con la palabra reservada var. Lo que sucede es que las declaraciones de variables y funciones son asignadas en memoria durante la fase de compilación, pero quedan exactamente en dónde se han escrito en el código.

Coerción

La coerción consiste en transformar de un tipo de dato a otro de una variable. Existen dos tipos de coerción: implícita y explícita.

La coerción implícita es cuando el lenguaje JavaScript ayuda con la transformación de tipos. Esto ocurre en operaciones de diferentes tipos, ya que es un lenguaje débil y dinámicamente tipado. Esto implica que el motor de JavaScript lo interpreta a su manera y podría arrojar un valor que puede ser erróneo, por lo cual es importante tener cuidado.

La coerción explícita es cuando se obliga a que transforme un tipo de dato. Para realizar estas transformaciones se utilizan funciones como: Number(), String() y Boolean(); parar convertir un dato a tipo número, string y lógico respectivamente.
