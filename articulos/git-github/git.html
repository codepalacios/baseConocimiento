Git y GitHub

¿Qué es Git?

Git es un sistema de control de versiones distribuido, deseñado por Linus Torvalds. Está pensado en la eficiencia y la confiabilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran número de archivos de código fuente. Git está optimizado para guardar todos esto cambios de forma atómica e incremental.

¿Qué es un sistema de control de versiones?

El SCV o VCS (por sus siglas en inglés) es un sistema que registra los cambios realizados sobre un archivo o conjunto de archivos a lo largo del tiempo, de modo que se pueda llevar el historial del ciclo de vida de un proyecto, comparar cambios a lo largo del tiempo, ver quién los realizó o revertir el proyecto entero a un estado anterior.

¿Qué es GitHub?

Es una plataforma de desarrollo colaborativo para alojar proyectos utilizando el sistema de control de versiones de Git. Se emplea principalmente para la creación de código fuente de programas de computadora. GitHub puede considerarse como la red social para los pragramadores y en muchos casos es visto como tu curriculum vitae, pues aquí guardas tu portafolio de proyectos de programación.

Ciclo de vida o estados de los archivos en Git

Cuando se trabaja con Git los archivos pueden vivir y moverse en 4 diferentes estados (con repositorios remotos pueden ser más estados):

  • Archivos Tracked: son archivos que viven dentro de Git, no tienen cambios pendientes y sus últimas actualizaciones han sigo guardadas en el repositorio gracias a los comandos git add y git commit.
  • Archivos Staged: son archivos en Staging. Viven dentro de Git y hay registro de ellos porque han sido afectados por el comando git add, aunque no sus últimos cambios, pero todavía no han sido guardados definitivamente en el repositorio por que falta ejecutar el comando git commit.
  • Archivos Unstaged: entendiéndolos como archivos "Tracked pero Unstaged". Son archivos que viven dentro de Git, pero no han sido afectados por el comando git add ni mucho menos por git commit. Git tiene un registro de estos archivos, pero está desactualizado, sus últimas versiones solo están guardadas en el disco duro.
  • Archivos Untraked: son archivos que no viven dentro de Git, solo en el disco duro. Nunca han sido afectados por git add, así que Git no tiene registro de su existencia. Existe un caso muy raro donde los archivos tiene dos estados al mismo tiempo: staged y untraked. Esto pasa cuando se guardan los cambios en el área de Staging (con el comando git add), pero antes de hacer commit para guardar los cambios en el repositorio se realizan nuevos cambios que todavía no han sigo guardados en el área de Staging (pero todo sigue funcionando igual).
