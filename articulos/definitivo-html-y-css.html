Web Developer Fundamentals

Existen tres estándares importantes para generar productos web. Esto son los lenguajes o estándares que entiende todo navegador web:

  • HTML: estructura de los documentos que se visualizan en los navegadores (Chrome, Edge, Firefox, Safari, Opera, entre otros).
  • CSS: es el lenguaje con el que le damos estilos a todo el contenido previamente etiquetado con HTML.
  • JavaScript: es el lenguaje que se emplea para crear o brindad a las páginas web la capacidad de interactuar con el usuaro y los recursos asociados a una página web.

¿Qué es el Frontend?

Frontend es la parte de un programa o dispositivo a la que un usuario puede acceder directamente. Son todas las tecnologías de diseńo y desarrollo eb que corren en el navegador y que se encargan de la interactividad con los usuarios.

Un programador Frontend debe saber de código que entienda el navegador (HTML, CSS y JavaScript) para poder usar algunos frameworks o librerías que expanden sus capacidades para crear cualquier tipo de interfaces de usuraios.

¿Qué es Backend?

Backend en programación corresponde al lado opuesto a un Front-end en un sitio web o aplicación, ya que el Backend trabaja en el lado del cliente. Es el responsable de manejar toda la lógica que hay detrás de una petición dad por el navegador hacia el servidor.

Una carecterística que lo diferencia del Frontend es que no tiene estándares, puesto que tiene varios lenguajes de programación (Node.js, Python, PHP, Ruby, GO, Java, .NET entre otros) con los que debe trabajar. Cada uno de estos lenguajes tiene su propios frameworks como:

  • Django (Python)
  • Laravel (PHP)
  • Rails (Ruby)
  • Express (JavaScript)
  • Spring (Java)

El Backend también tiene en cuenta la infraestructura donde va a realizar el deploy de su aplicación (esto también puede ser tarea de un DevOps, un perfil dedicado a la infraestructura), con tecnologías como:

  • Google Cloud
  • DigitalOcena
  • AWS
  • Heroku, entre otras.

Qué es deploy?

Deploy es yb término famoso entre los desarrolladores web. Puede significar muchas cosas, dependiendo del ambiente y de la tecnología usada. Sin embargo, los significados que más se refieren a la práctica y pueden resumir su función son: implantar, colocar en posición habilitar para uso o, simplemente, publicar.

Por último, entramos en base de datos, que son las encargadas de almacenar toda la información del proyecto Los principales tipos son:

  • Bases de datos relacionales (como MySQL)
  • Base de datos no relacionales (como mongoDB)
